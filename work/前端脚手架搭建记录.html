前端脚手架搭建 #

TIP

脚手架的搭建的模仿 vite 的官方脚手架 create-vite 的实现的

背景: 项目组对老项目进行功能重构，部分功能独立出来成为一个单独的应用，我在搭建项目的时候突发奇想的想搞一个脚手架出来，减少以后在创建新项目时的一些基建操作

初始化 #

shell
mkdir hf-vite-cli && cd hf-vite-cli
pnpm init
pnpm add prompts kolorist @types/node -D
  • prompts 用于和终端交互的
  • kolorist 终端文字样式

// index.js
import prompts from 'prompts' // 终端交互
import { reset } from 'kolorist' // 终端文字显示效果

const createProject = async () => {
  try {
    const res = await prompts([

        message: reset('Project name:'), // 效果类型文本框的 placeholder
        initial: 'hf-project', // 默认值
      },
    ])
    console.log(res)
  } catch (err) {
    throw new Error('出错了.....')
  }
}

代码结构优化 #

到这终端交互部分就差不多了，在对这部分代码进行优化下（比较简单，直接放代码）：

  • 框架类型数据结构化，便于之后添加其他模板

// constant.js
export const framework = [
  {
    value: 'vue',
    title: 'vue3 + ts',
    ui: [
      { value: 'antd', title: 'Ant-design-vue' },
      { value: 'element', title: 'Element-plus' },
    ],
  },
  {
    value: 'react',
    title: 'react18 + ts',
    ui: [
      { value: 'antd5', title: 'Ant-design 5' },
      { value: 'arco', title: 'Arco design' },
    ],
  },
]
  • 交互逻辑单独提取
javascript
// utils.js
import { framework } from './constant'

export const getFrameworkList = () => {
  return framework.map((item) => ({
    value: item.value,
    title: item.title,
  }))
}

export const getFrameworkUiList = (type) => {
  return framework.find((item) => item.value === type).ui
}
  • 修改 index.js
javascript
// index.js
import prompts from 'prompts'
import { reset } from 'kolorist'
import { getFrameworkList, getFrameworkUiList } from './utils'

const createProject = async () => {
  try {
    const res = await prompts([
      {
        type: 'text',
        name: 'projectName',
        message: reset('请输入项目名称'),
        initial: 'hf-project',
      },
      {
        type: 'select',
        name: 'framework',
        message: reset('请选择框架'),
        initial: 0,
        choices: getFrameworkList(),
      },
      {
        type: 'select',
        name: 'frameworkUI',
        message: reset('请选择相应的 UI 库'),
        initial: 0,
        choices: (framework) => getFrameworkUiList(framework),
      },
    ])
    console.log(res)
    // 下载模板
    // downloadTemplate(res)
  } catch (err) {
    throw new Error('出错了.....')
  }
}

模板下载实现 #

模板下载也是比较简单的，首先是我们的文件夹目录设置,大致如下：

javascript
- node_modules
- src
	- index.js  // 入口文件
	- utils.js  // 工具函数
	- constant.js // 静态变量
  - download.js // 模板下载
- template-vue-antd
- template-vue-element
- template-react-antd
- template-react-arco
- package.json
- .gitignore

template-xxx-xxx : 都是一个完整的项目，下载到本地后并安装依赖后即可运行，模板下载就是根据前面终端交互获取到的信息，把提前写好的模板下载在用户本地

具体逻辑：

  • 首先确定我们输入的文件夹名称（默认 hf-project）在当前执行路径上 （process.cwd()）是否已存在，如果存在，则报错提示目标文件夹已存在 —— fs.existDirSync
  • 如果当前执行脚本的路径上没有目标文件夹，则新建目标文件夹 —— fs.mkdirSync
  • 读取模板项目的所有文件
    • 判断是否是文件夹：fs.statSync().isDirectory()
      • 如果是文件夹，递归操作；
        • 读取文件夹： fs.readFileSync
      • 如果是文件，直接拷贝到前面新建的文件夹里面
        • 复制文件：fs.copyFileSync
  • 特殊文件
    • _gitignore: 由于模板要上传到仓库中，所以 .gitignore 文件名被修改了，复制的时候需要改回去
    • package.json: 里面的 name 字段的值需要换成用户输入的项目名

这部分代码量有点多就不放到这里面了，想看源码可以看这里：hf-vite-cli
